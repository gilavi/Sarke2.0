## 1 — Onboarding / Login

Files: [login.tsx](app/(auth)/login.tsx), [forgot.tsx](app/(auth)/forgot.tsx), [reset.tsx](app/(auth)/reset.tsx), [verify-email.tsx](app/(auth)/verify-email.tsx), [_layout.tsx](app/(auth)/_layout.tsx), [app/index.tsx](app/index.tsx), [lib/session.tsx](lib/session.tsx)

### P3 — Deep-link reset code dedup is in-memory only — `app/_layout.tsx:26`
**Repro:** Tap reset link in email → app cold-starts and exchanges code → user backgrounds and the OS kills the app → tap link again from email.
**Cause:** The `exchangedCodes` Set guarding against double-exchange lives in JS module memory and is reset on cold launch. A second cold launch with the same `?code=` re-attempts exchange.
**Fix:** Persist consumed codes (e.g. an `AsyncStorage` Set with a 24h TTL). Currently low risk because the second exchange fails gracefully and routes to forgot screen with `err=expired` — annoying UX but no security or data issue.

**✓ Auth — 1 P3.**

## 5 — Add/Remove მონაწილეები (Bottom Sheet)

Files: [components/CrewSection.tsx](components/CrewSection.tsx), [components/BottomSheet.tsx](components/BottomSheet.tsx)

### P2 — Bottom sheet dismisses before parent's async `onChange` resolves — `components/CrewSection.tsx:58-62`
**Repro:** Tap "შენახვა" on the crew member form on a slow connection.
**Cause:** Sheet calls `dismiss()` immediately and the parent runs `onChange(...)` in the background. If `onChange` rejects (server error, optimistic rollback), the sheet is already gone and the user sees no error context — only the row reappearing/disappearing in the list.
**Fix:** Add a local `submitting` flag; keep the sheet open with a spinner while `onChange` resolves; only `dismiss()` on success. Toast on error stays usable.

### P3 — Crew member ID generation can collide on `Math.random()` fallback — `components/CrewSection.tsx:210-213`
**Repro:** On a device where `crypto.randomUUID` is unavailable, add two crew members in the same millisecond.
**Cause:** Fallback combines `Date.now()` + `Math.random()`, which is unique enough in practice but theoretically collidable. Real risk is very low — `expo-crypto` polyfills randomUUID, and humans don't add two members in the same ms.
**Fix:** Either drop the fallback (rely on `expo-crypto`'s polyfill in `lib/polyfills.ts`) or include a monotonic counter. Could also be left as-is.

**⚠ Crew sheet — 1 P2 + 1 P3.**

## 6 — Start New კითხვარი

Files: [app/template/[id]/start.tsx](app/template/%5Bid%5D/start.tsx), [app/(tabs)/home.tsx](app/(tabs)/home.tsx) (`ProjectPickerSheet`)

### P2 — Stale `pickedProjectId` race in template picker — `app/(tabs)/home.tsx:569`
**Repro:** Open the project picker → pick a project → on the template list, swipe back to the project list (resets `pickedProjectId` to `null`) → before the swipe completes, tap a template.
**Cause:** Template `onPress` is `() => pickedProjectId && void startInspection(pickedProjectId, t.id)`. The `pickedProjectId` check happens at press time but the closure captures whatever is in state at that moment; with React's batched state updates, the value can be stale.
**Fix:** Guard inside `startInspection` itself — re-validate the project still exists, or pass the project id by value into a stable callback rather than closure-captured state.

**⚠ Start new კითხვარი — 1 P2.**

## 9–12 — Wizard Sections (Yes/No, Component Grid, Photos, Free Text)

File: [app/inspections/[id]/wizard.tsx](app/inspections/%5Bid%5D/wizard.tsx)

### P1 — Photo picker callback bus has no handle/ID — `lib/photoPickerBus.ts:15`
**Repro:** On a photo question, tap the photo button → swipe back from the picker without selecting → tap photo again on the *next* question → select an image.
**Cause:** `setPhotoPickerCallback` overwrites a single module-level `pending` callback with no id/handle. If the first caller never resolved (because the user swiped back instead of selecting/cancelling), the second caller's setter overwrites it. The first caller's URI is fine here (it gets set to null on cancel), but the *image* selected for question 2 is delivered via the latest callback — which is question 2's, so this case actually works. **But:** if the user on iOS receives the image-selection result while a stale `pending` from a prior cold path is still set, the resolution races. This is fragile.
**Fix:** Make `setPhotoPickerCallback` return a token; `resolvePhotoPicker` accepts the token and only fires if it matches the current pending. Also, the picker screen should call `cancelPhotoPicker()` on its unmount/back-nav to clear stale state.

### P2 — Wizard freetext debounce flush on unmount is fire-and-forget — `app/inspections/[id]/wizard.tsx:986-995`
**Repro:** Type into a freetext question → swipe back to project before 500ms debounce fires → network is offline at that moment.
**Cause:** Unmount effect calls `onCommit()` synchronously without awaiting; if the underlying `patchAnswer` rejects, the rejection is unhandled and the user's text is silently lost (it was never flushed to local storage).
**Fix:** Either await the flush before unmount completes (not always feasible in React lifecycle), or — better — flush to AsyncStorage synchronously before kicking off the debounce/network step, so power-off / app-kill doesn't lose the edit.

### P2 — Step transition animations re-run on multi-step jumps — `app/inspections/[id]/wizard.tsx:173-181`
**Repro:** Tap back arrow rapidly 5+ times.
**Cause:** Every `setStepIndex` triggers a re-animation regardless of whether `Math.abs(next - prev) > 1`. UX is sluggish, not broken.
**Fix:** If the diff is >1, jump without animation (`stepAnim.setValue(...)` instead of `Animated.timing`).

**⚠ Wizard — 1 P1 + 2 P2.**

> Note: Agent 2 also flagged a `harness_name` field-loss claim in the wizard's `stripServerFields` merge. **Discarded** on verification — `stripServerFields` only strips `status`, `completed_at`, `updated_at`, `created_at`, `user_id` ([lib/offline.tsx:35-49](lib/offline.tsx)). `harness_name` is preserved correctly.

## 14 — PDF Generation

Files: [lib/pdf.ts](lib/pdf.ts), [lib/sharePdf.ts](lib/sharePdf.ts)

### P3 — `created_at` formatting has no fallback — `lib/pdf.ts:57`
**Repro:** Generate PDF for a row whose `created_at` is somehow null/undefined (shouldn't happen given DB constraints — defensive only).
**Cause:** `new Date(undefined).toLocaleDateString('ka')` returns `'Invalid Date'`. Will appear in the PDF header.
**Fix:** `const date = q.created_at ? new Date(q.created_at).toLocaleDateString('ka') : '—';`

### P3 — `sharePdf` non-null assertion on `documentDirectory` — `lib/sharePdf.ts:17`
**Repro:** Theoretical — both `cacheDirectory` and `documentDirectory` undefined.
**Cause:** `(FileSystem.cacheDirectory ?? FileSystem.documentDirectory!)` — second fallback uses `!` non-null assertion.
**Fix:** Replace with explicit guard + meaningful error: `if (!base) throw new Error('No filesystem dir available')`.

> Both are defensive-only; in practice these dirs are defined on iOS and Android.

**⚠ PDF — 2 P3.**

### P2 — Completion timestamp displayed in device timezone — `app/inspections/[id]/done.tsx:108`, `app/history.tsx:160`
**Repro:** Inspection completed by a user in UTC+4, viewed by another user (or same user travelling) in a different timezone.
**Cause:** `.toLocaleString('ka')` formats with device timezone but the source value is the raw ISO from the server. There's no canonical timezone associated with the inspection.
**Fix:** If the business logic considers an inspection "owned" by a project location, format using that project's timezone. Otherwise, display in UTC with a clear suffix, or accept the device-timezone behavior and document it.

## 16 — Settings / Profile (More tab)

File: [app/(tabs)/more.tsx](app/(tabs)/more.tsx)

### P1 — Sign-out has no loading guard — `app/(tabs)/more.tsx:192-193`
**Repro:** Tap "გასვლა" rapidly twice, or tap and immediately tap a section row.
**Cause:** `onPress={signOut}` is fire-and-forget — no `loading`/`disabled` state. Rapid taps fire `signOut()` twice; navigation during the in-flight signOut can leave the user on a screen requiring a session that's mid-teardown.
**Fix:** Wrap with `const [signingOut, setSigningOut] = useState(false)` and disable the button while in flight. Pattern is already used elsewhere (e.g. `Button` accepts `loading`).

### P3 — Locale code inconsistency — `app/qualifications/index.tsx:130`
**Repro:** Cosmetic — date formatting in qualifications uses `'ka'` while [home.tsx:679](app/(tabs)/home.tsx) uses `'ka-GE'`.
**Cause:** Inconsistent locale strings. Both work in modern Intl, but `'ka-GE'` is the canonical BCP-47 form and behaves more predictably across older devices.
**Fix:** Standardize on `'ka-GE'` everywhere. Also consider centralizing in `lib/locale.ts`.

**⚠ More — 1 P1 + 1 P3.**

1. **"`UploadedFilesSection` reduce on empty array crashes"** — Component already correctly guards with `count > 0` before accessing `files[0]` ([components/UploadedFilesSection.tsx:14-16](components/UploadedFilesSection.tsx)).
2. **"Wizard's `stripServerFields` drops `harness_name`"** — `stripServerFields` only strips server-canonical fields (`status`, `completed_at`, `updated_at`, `created_at`, `user_id`) per [lib/offline.tsx:35-49](lib/offline.tsx). `harness_name` is preserved.
3. **"`CreateProjectSheet` is missing the address Input"** — Reframed as a P3 UX inconsistency. The address input *is* present (rendered inside `MapPicker`); the inconsistency is between Home and Projects forms, not a missing field.
4. **"Photo upload skipProcessing rotates portrait photos"** — Not verified; both Hermes/`expo-image-picker` and the FileSystem upload path were recently overhauled (see commit `feb13af`). Skipped.
