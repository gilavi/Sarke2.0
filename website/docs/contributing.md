# Contributing

## Editing these docs

Markdown lives under `website/docs/`. Files are rendered by Docusaurus 3.

- `docs/intro.md` — landing page (its `slug: /` makes it the site root).
- `docs/routes/*.md` — per-screen reference.
- `docs/components.md`, `docs/lib.md` — UI + lib reference.
- `docs/data-model.md`, `docs/database/*.md` — schema.
- Sidebar order is auto-generated by Docusaurus from `sidebar_position` frontmatter.

To preview locally:

```sh
cd website
npm install
npm run start
```

## Mermaid diagrams

```mdx
```mermaid
flowchart LR
  A --> B
```
```

Mermaid is enabled via `markdown.mermaid: true` in `docusaurus.config.ts` and the `@docusaurus/theme-mermaid` theme.

## Refreshing the OpenAPI spec

When the Supabase schema changes:

```sh
node website/scripts/fetch-openapi.mjs
```

…then commit the updated `website/static/openapi.json`.

## Style

- Don't repeat code that's already in `types/models.ts` or migrations — link to it.
- Keep migration changelog entries to one line of summary plus a "what changed in TS" sentence if it had a model-side rename.
- Don't document private internal helpers in `lib.md` — only files that are referenced from screens or other lib files.
