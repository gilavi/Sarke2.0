# Inspection Routes

The inspection flow is the core of the app: pick a template, walk through the questions, sign, generate a certificate.

| Path | File | Purpose |
| --- | --- | --- |
| `/templates` | `app/templates.tsx` | Pick a template to start from |
| `/template/[id]/start` | `app/template/[id]/start.tsx` | Choose target project + project_item, create the inspection draft |
| `/inspections/[id]/wizard` | `app/inspections/[id]/wizard.tsx` | Step-by-step questionnaire — one question per screen, with photo upload, measurements, grids, freetext |
| `/inspections/[id]/done` | `app/inspections/[id]/done.tsx` | Conclusion + safe-for-use toggle, signing, PDF generation |
| `/signature` | `app/signature.tsx` | Reusable signature-capture screen (used by the done step + remote signing) |
| `/photo-picker` | `app/photo-picker.tsx` | Modal photo picker (camera / library / files) |

## Wizard mechanics

- Questions are loaded from `questions` table for the inspection's template.
- Answers are persisted on every "next" tap (one row per `(inspection_id, question_id)`).
- Photos are uploaded immediately via `FileSystem.uploadAsync` and rows inserted into `answer_photos`.
- The wizard is resumable — re-entering an inspection draft restores the last answered question.

## Done step

- `is_safe_for_use` (boolean) and `conclusion_text` are required to mark complete.
- Required signers come from `templates.required_signer_roles` joined with `project_signers`; missing signers are gathered inline (modal) or invited via SMS / email (remote signing).
- On "complete", the inspection status flips to `completed`, `completed_at` is stamped, and a `Certificate` row is generated by [`lib/pdf.ts`](../lib.md#pdfts).

After completion the inspection is **frozen** by DB triggers (migrations `0008` and `0010`) — no edits, no answer changes, no status reverts.

See [Signing flow](../signing-flow.md) and [PDF generation](../pdf-generation.md).
