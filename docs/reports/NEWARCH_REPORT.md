# New Architecture — Diagnosis Report

**STATUS: ON (verified via `app.json` + react-native-reanimated 4.x runtime requirement).**

The Sarke 2.0 mobile app is running on React Native's New Architecture (Fabric renderer + TurboModules) on `main`. No changes were made by this report.

---

## Evidence

1. **`app.json` declares it explicitly.** Line 10:
   ```json
   "expo": {
     ...
     "newArchEnabled": true,
     ...
   }
   ```
   No per-platform overrides (`expo.ios.newArchEnabled`, `expo.android.newArchEnabled`) exist, so the top-level flag is the effective value on both platforms.

2. **`react-native-reanimated` is on 4.x** (`~4.1.1` per `package.json`). Reanimated 4 only runs on the New Architecture — if New Arch were off, the app would crash at boot when worklets initialise. The app boots cleanly in Expo Go (verified locally throughout the v1 + v2 refactor sessions), which is independent evidence that New Arch is active at runtime regardless of what the flag says.

3. **Expo SDK 54.0.34** (per `package.json`). SDK 54 ships with New Arch as the default for new projects and supports it on both platforms. Matches the README.

4. **Managed workflow — no `ios/` or `android/` folders in the repo.** The `ios/` folder mentioned in the README's Directory Layout table refers to a legacy native scaffold that is **not present on `main`** (the live native port lives on the [`ios-legacy`](https://github.com/gilavi/sarke2.0/tree/ios-legacy) branch). So there are no `ios/Podfile.properties.json` or `android/gradle.properties` overrides to inspect; Expo's prebuild generates them on demand from the `app.json` flag.

## Manual verification steps

If you want to re-confirm New Arch is active end-to-end after pulling these doc changes:

1. **Clear caches and start the dev server:**
   ```sh
   npx expo start --clear
   ```

2. **Smoke test the canonical flow on a real device or simulator** (Reanimated 4 surfaces are the load-bearing check — anything animated would break if New Arch flipped off):
   - Log in.
   - Open a project.
   - Step through one inspection wizard question (gesture handler + keyboard controller).
   - Generate a PDF preview (`expo-print`).
   - Capture a signature (`react-native-signature-canvas`).
   - Open the project map hero (`react-native-maps`).

3. **Rollback (if you need to revert this report and any subsequent doc changes):**
   ```sh
   git reset --hard pre-newarch-and-docs
   ```
   The `pre-newarch-and-docs` tag was created before this report.
