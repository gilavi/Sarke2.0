# Payments — Hubble Pro subscriptions

Single source of truth for the BOG payment / subscription flow shared by the mobile app and the `web-app/` dashboard.

## Pricing

- **Hubble Pro:** ₾1 / month (testing rate as of 2026-06-11 — change back to ₾19 before full launch). Charged via Bank of Georgia (BOG) e-commerce.
- **Free tier:** `PDF_FREE_LIMIT = 30` lifetime PDFs (intentionally generous during soft launch — tighten when BOG payment is fully approved).

## Backend

### `users` columns

| Column | Type | Purpose |
|---|---|---|
| `pdf_count` | int | PDFs generated by this user (free-tier counter). |
| `subscription_status` | `'free' \| 'active' \| 'expired'` | Current tier. |
| `subscription_expires_at` | timestamptz | When the active period ends. |
| `subscription_cancelled_at` | timestamptz | Set by `cancel_subscription`. Access continues until `expires_at`. |
| `bog_card_token` | text | BOG card token captured at first payment (for future auto-renewal — not yet wired). |

### RPCs

- **`increment_pdf_count(user_id uuid) -> jsonb`** — atomic. Auto-expires lapsed subscriptions. Returns `{ allowed, count, limit }`. Active subscribers bypass the limit. Defined in [supabase/migrations/0029_subscription_unlimited.sql](../supabase/migrations/0029_subscription_unlimited.sql).
- **`cancel_subscription(user_id uuid) -> jsonb`** — idempotent. Sets `subscription_cancelled_at = now()`. Returns `{ cancelled, active_until }`. Defined in [supabase/migrations/0031_subscription_cancel_and_history.sql](../supabase/migrations/0031_subscription_cancel_and_history.sql).

### `payment_records` table

Written by `bog-payment-callback` for every BOG webhook (success + failure + refund). RLS: select-own. Status enum maps from BOG `order_status.key`:

| BOG status | `payment_records.status` |
|---|---|
| `completed` | `success` |
| `refunded`, `refunded_partially` | `refunded` |
| `rejected`, `declined`, `expired` | `failed` |
| (anything else) | `pending` |

Unique index on `(bog_order_id, status)` deduplicates webhook retries.

### Edge Functions

- **`create-bog-order`** ([source](../supabase/functions/create-bog-order/index.ts)) — auth'd. Validates redirect URLs (allows `sarke2://`, `https://hubble.ge/...`, and the legacy `https://gilavi.github.io/Sarke2.0/...` which 301s to the CNAME). Calls BOG, returns `{ order_id, redirect_url }`.
- **`bog-payment-callback`** ([source](../supabase/functions/bog-payment-callback/index.ts)) — public webhook (BOG calls it). Re-verifies status server-side, upserts a `payment_records` row, and on `completed` flips the user to `active` for 30 days, clearing `subscription_cancelled_at`.

## Flow

> **Mobile sells nothing (2026-06-12).** Apple guideline 3.1.1 / Google Play policy forbid
> non-IAP purchase flows for digital subscriptions, so the mobile app contains **zero purchase
> UI** — no prices, no payment links, no subscribe buttons, no payment deep links. Purchasing is
> **web-only**. Mobile shows subscription *status* (allowed) and unlocks automatically when
> `users.subscription_status` flips to `active` (`lib/usePdfUsage.ts` re-reads it). At the free
> limit, mobile shows the neutral `SubscriptionNotice` (no price/URL/CTA).

```
                          web /account
                               │
                               ▼
        /subscribe (web only — mobile has no purchase entry point)
                  │
                  ▼
        invoke create-bog-order ──► BOG redirect URL
                                          │
                                          ▼
                              [user pays at BOG]
                                          │
                ┌─────────────────────────┴────────────────────────┐
                ▼                                                  ▼
   /subscribe/success                                  /subscribe/fail
   (invalidates pdf-usage                              (web shows retry CTA)
    + payment-history caches on web)
                │
                │  (asynchronously, BOG hits the webhook)
                ▼
   bog-payment-callback verifies + activates subscription
```

### Triggering the paywall on web (when web PDF generation lands)

```ts
import { checkAndIncrementPdfCount, PdfLimitReachedError } from '@/lib/pdfGate';

try {
  await checkAndIncrementPdfCount(userId);
  await generatePdf(...);
} catch (e) {
  if (e instanceof PdfLimitReachedError) setPaywallOpen(true);
  else throw e;
}
```

Same shape as [app/reports/[id]/success.tsx](../app/reports/%5Bid%5D/success.tsx).

## Secrets & deployment

BOG credentials live **only** in Supabase Edge Function secrets (Dashboard → project → Edge Functions → Secrets, or `supabase secrets set`). Never in the repo, `web-app/` env files, or the app bundle — BOG auth is server-side OAuth (`client_id` + `client_secret` → Basic auth token request), so the client never needs a key.

| Secret | Value |
|---|---|
| `BOG_CLIENT_ID` / `BOG_CLIENT_SECRET` | BOG e-commerce OAuth pair (sandbox or production). |
| `BOG_ENV` | `production` for live keys; anything else (or unset) selects the sandbox BOG endpoints. |
| `BOG_CALLBACK_URL` | `https://<project-ref>.supabase.co/functions/v1/bog-payment-callback` |

For local dev, put sandbox values in `supabase/.env.local` (gitignored).

No CI deploys edge functions — after changing function code, deploy manually:

```sh
supabase functions deploy create-bog-order
supabase functions deploy bog-payment-callback
```

**Gotcha (2026-06):** payments were broken because the deployed `create-bog-order` predated the hubble.ge rebrand — its redirect-URL allowlist rejected the new domain with `400 invalid redirect url`. If the subscribe page's domain ever changes again, update `ALLOWED_PREFIXES` in the function **and redeploy it**.
