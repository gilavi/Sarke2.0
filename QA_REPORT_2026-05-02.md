## Bug 1 — `FloatingLabelInput` `style` prop applied to inner bordered container · FIXED

**File:** `components/inputs/FloatingLabelInput.tsx`

**Symptom:** Callers passing `style={{ marginBottom: 0 }}` or `style={{ flex: 1 }}` had no effect — the value was silently overridden by `styles.wrapper`. Callers passing `style={{ borderRadius: 20 }}` would affect the inner bordered box rather than the outer wrapper.

**Root cause:** `style` was included in the `containerStyle` array (the inner bordered `<View>`) rather than the outer wrapper `<View style={styles.wrapper}>`.

**Fix:**
```diff
- const containerStyle = [..., style];
- return <View style={styles.wrapper}><View style={containerStyle}>
+ const containerStyle = [...]; // style removed
+ return <View style={[styles.wrapper, style]}><View style={containerStyle}>
```

**Severity:** P2 — layout-only, no data loss, but causes invisible layout bugs for any caller that relies on spacing/sizing overrides.

## Bug 2 — `HarnessListFlow` photo thumbs spin forever for server-stored photos · FIXED

**File:** `components/HarnessListFlow.tsx` (`CellPhotoThumb`)

**Symptom:** Harness inspection photos taken in a previous session (stored in Supabase Storage as bucket paths) show an `ActivityIndicator` spinner indefinitely. Only photos taken in the current offline session (local `file://` URIs) displayed correctly.

**Root cause:** `CellPhotoThumb` initialised `uri` as `null` for non-local paths and never fetched the display URL:
```ts
const [uri] = useState<string | null>(isLocal ? photo.storage_path : null);
// ^ uri stays null forever for storage-path photos
```

**Fix:** Added a `useEffect` that calls `getStorageImageDisplayUrl` for non-local paths:
```ts
const [uri, setUri] = useState<string | null>(isLocal ? photo.storage_path : null);
useEffect(() => {
  if (isLocal) return;
  let cancelled = false;
  getStorageImageDisplayUrl(STORAGE_BUCKETS.answerPhotos, photo.storage_path)
    .then(url => { if (!cancelled) setUri(url); })
    .catch(() => {});
  return () => { cancelled = true; };
}, [photo.storage_path, isLocal]);
```

**Severity:** P1 — any harness inspection with photos loaded from a previous session showed broken UI.

## Bug 3 — Duplicate label on harness name field in `ConclusionStep` · FIXED

**File:** `app/inspections/[id]/wizard.tsx` (`ConclusionStep`)

**Symptom:** The harness name input showed the text "ღვედის დასახელება" twice — once as a plain `<Text>` label above the field, and again as `FloatingLabelInput`'s animated floating label inside the field.

**Root cause:** A separate `<Text>` label block had been added around `<FloatingLabelInput label="ღვედის დასახელება" ...>`, not realising `FloatingLabelInput` renders its own label.

**Fix:** Removed the outer `<View>/<Text>` wrapper, leaving only `<FloatingLabelInput>` which carries its own built-in label.

**Severity:** P2 — visual noise, not a functional breakage.

## Bug 5 — `home.tsx` called `toast.error()` with `toast` undefined · FIXED

**File:** `app/(tabs)/home.tsx`

**Symptom:** On the Home screen, the code path that handles a missing template (error branch when `templatesApi.getById` returns null) called `toast.error(...)`, but `const toast = useToast()` was absent from the component. This would throw a runtime `TypeError: Cannot read properties of undefined` the first time a user tapped a briefing or report whose template could not be found.

**Root cause:** `const toast = useToast();` was missing — likely dropped during a merge/rebase conflict resolution.

**Fix:** Added `const toast = useToast();` after `const qc = useQueryClient();` in `HomeScreen`.

**Severity:** P1 — hard crash on the home screen for affected users.

## Bug 6 — Mock `Project` objects missing required `contact_phone` field · FIXED

**File:** `lib/services.mock.ts`

**Symptom:** TypeScript error `Property 'contact_phone' is missing in type '...' but required in type 'Project'` when running in mock/dev mode. Mock projects (`proj1`, `proj2`, and the dynamically created project in `projectsApi.create`) were not updated when `contact_phone: string | null` was added to the `Project` interface in `types/models.ts`.

**Fix:** Added `contact_phone: null` to all three mock `Project` objects.

**Severity:** P2 — compile-time error; would prevent running the app in mock mode.
